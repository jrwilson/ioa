3 Reference

— Class: ioa::automaton

The base class for all automata that implements the system action interface which allows an automaton to request system actions and receive system action results. All automata must publicly inherit from ioa::automaton. From <ioa/automaton.hpp>.

— Class: ioa::global_fifo_scheduler

A single-threaded scheduler that implements the first-in/first-out (FIFO) policy. From <ioa/global_fifo_scheduler.hpp>.

— Function: template <class T> void ioa::run (scheduler_interface& sched, std::auto_ptr<typed_allocator_interface<T> > allocator)

Starts the scheduler sched with the root automaton produced by allocator. From <ioa/scheduler.hpp>.

— Function: template <class T, A0, ...> std::auto_ptr<typed_allocator_interface<T, A0, ...> > make_allocator (A0 a0, ...)

A set of helper functions for allocating allocators. This set of functions takes advantage of the automatic type deduction rules of C++ to avoid repeating type information. From <ioa/allocator.hpp>.
